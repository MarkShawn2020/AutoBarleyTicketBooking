## 抢票系统的设计与实现
计划分客户端和服务端分步进行。
### 服务端
对于服务端，直接使用postman进行接口分析，然后开发python或者js脚本。

### 客户端
- 首先基于oliwang的油猴脚本进行代码优化与二度开发，然后重新提交到油猴测试。
- 如果测试通过，则即可以选择直接使用油猴脚本，也可以考虑在自建服务器上搭建一个大麦网的沙盒，方便客户直接下单，免去用户手动配置油猴的琐碎过程。
